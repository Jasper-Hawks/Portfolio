Introduction to my Metadata Tag Editor, MetaMusic

08/30/2022

I hate music streaming services.

I hate that I don't have the physical copies of the songs and albums that I listen to. I enjoyed when DVDs and vinyls were the norm where the scratches and imperfections told as much a story as the music contained within them. Yet, I see the convenience and reliability of digital music.

Music that won't wear down with age. That sounds as if the artist just recorded it and put it to a physical medium, but even in this case I'm a special case.

I like to modify tags and album covers and use music files in scripts and other programs. This usually ends up with me deleting Metadata and it would be helpful if there was tool to to allieviate my unique problem.

This is why I created MetaMusic. MetaMusic is a Python3 script that applies metadata to files using an unofficial Youtube Music API. It can apply metadata to entire albums or single files and accepts stdin.

MetaMusic is a script-friendly metadata tag editor that is simple, extensible, and fast.

MetaMusic does this by searching Youtube Music for the name of the file or the name of the directory, finding the song or album, then applying the data to the file(s). It does all of this within seconds even for the largest of albums all while outputting status codes so you know what songs have had data applied and which have not.

MetaMusic also has a myriad of flags in order to help the user find the music they would like to tag. Using the help flag you can see all the helpful flags that MetaMusic comes packaged with. All while adhering to the Unix Philosiphy of doing one thing and doing that thing well.

If you think you can find some use out of MetaMusic consider downloading it from Github and trying it. If you really enjoy it consider emailing me a nice message or support me on Ko-fi.
